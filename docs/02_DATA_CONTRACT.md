## Соглашения по кодам карточек

Формат: `<ГРУППА>-<НОМЕР>`, где ГРУППА — одно- или двухбуквенный русский идентификатор.

### Формула-7 (8 групп)

| Код | Группа | Размер | Роль |
|---|---|---|---|
| `СЧЖ-1..12` | Варианты счастья и организации жизни | 12 | вспомогательная |
| `Ц-1..9` | Цели труда | 9 | основная |
| `П-1..9` | Предмет труда | 9 | основная |
| `С-1..9` | Средства труда | 9 | основная |
| `У-1..9` | Условия и организация труда | 9 | основная |
| `О-1..9` | Особенности работы | 9 | основная |
| `К-1..9` | Коммуникации в труде | 9 | основная |
| `ВК-1..9` | Особенности обучения в вузе/колледже | 9 | финальная |

**Итого: 75 карточек.** Шесть «основных» групп (Ц, П, С, У, О, К) участвуют в правиле «хотя бы одна из каждой в формуле».

## Схема `shared-data/formula7/cards.json`

```json
{
  "meta": {
    "method": "F7",
    "formulaSize": 7,
    "mainGroups": ["Ц", "П", "С", "У", "О", "К"],
    "rankingOrder": ["СЧЖ", "Ц", "П", "С", "У", "О", "К", "ВК"],
    "groups": [
      {
        "key": "СЧЖ",
        "label": "СЧЖ",
        "name": "Варианты счастья и организации жизни",
        "role": "auxiliary",
        "intro": "Двенадцать образов жизни. ..."
      }
    ]
  },
  "cards": [
    {
      "code": "СЧЖ-1",
      "group": "СЧЖ",
      "method": "F7",
      "title": "Нормальная жизнь",
      "description": "Семья, дети, дом, дача, ..."
    }
  ]
}
```

### Поля карточки

| Поле | Тип | Обязательно | Примечание |
|---|---|---|---|
| `code` | `string` | да | Уникальный в пределах методики |
| `group` | `string` | да | Ключ группы (см. таблицы выше) |
| `method` | `"F7" \| "F5" \| "KCHG" \| "PEREKRESTOK"` | да | |
| `title` | `string` | да | Главная формулировка. Сохранять авторские формулировки дословно. |
| `description` | `string` | нет | Пояснение в скобках из книги. Не выдумывать. |
| `linked_Ch` | `int[]` | только для КЧГ К-карт | |
| `linked_G` | `int[]` | только для КЧГ К-карт | |

### Поля `GroupMeta`

| Поле | Тип | Примечание |
|---|---|---|
| `key` | `string` | используется в `cards[*].group` |
| `label` | `string` | короткая подпись, часто равна `key` |
| `name` | `string` | полное название для заголовков |
| `role` | `"auxiliary" \| "main" \| "final"` | влияет на UX (см. `F7Ranking.tsx`) |
| `intro` | `string` | текст над группой на экране ранжирования |

### Инварианты (должны держаться всегда)

- В F7 ровно 75 карточек, распределённые как 12+9+9+9+9+9+9+9.
- Коды уникальны внутри файла.
- `meta.mainGroups` содержит ровно шесть групп.
- `meta.rankingOrder` покрывает все `meta.groups`.
- `meta.formulaSize` совпадает с константой в коде (TS: `F7_FORMULA_SIZE`, Python: `F7.formula_size`).

Тест `test_f7_data_loads` в `core/tests/test_formula7.py` валидирует первые три инварианта.

## Схема `shared-data/formula7/provocations.json`

Плоский массив объектов `Provocation`:

```json
[
  {
    "id": "p-ts-1",
    "type": "card",
    "target": "Ц-1",
    "tone": "neutral",
    "text": "Оценивать и обследовать — это всегда смотреть сверху. Тебе нравится эта позиция?"
  },
  {
    "id": "pp-contra-1",
    "type": "pattern",
    "tone": "neutral",
    "trigger": { "kind": "contradiction", "cards": ["О-1", "О-5"] },
    "text": "Честность и риск одновременно — это путь следователя..."
  }
]
```

### Типы провокаций

| `type` | Триггер | `target` |
|---|---|---|
| `card` | открытие/выбор карточки | код карточки, например `"Ц-6"` |
| `group` | завершение ранжирования группы | ключ группы, например `"Ц"` |
| `formula` | фаза сборки формулы | нет |
| `molecule` | фаза сборки молекулы | нет |
| `pattern` | поведенческий паттерн, см. `trigger` | нет |

### Схема `trigger` (только для `type: "pattern"`)

Discriminated union по полю `kind`:

```ts
| { kind: "all_like_in_group"; group: string }
| { kind: "all_reject_in_group"; group: string }
| { kind: "many_flips"; count: number }
| { kind: "contradiction"; cards: [string, string] }
| { kind: "long_pause_on_card"; ms: number }
| { kind: "quick_pass_in_group"; group: string; avgMs: number }
```

### Правила формулировок

- **Дословность.** Авторские формулировки (с грубоватыми оттенками, как «торговля — это про деньги или про людей?») сохранять как есть. См. спеку §11, §17.
- **Один вопрос на провокацию.** Не «что ты думаешь и как это соотносится с...?» — а один ясный вопрос.
- **Не давать ответа** внутри провокации. Программа задаёт вопрос, не навязывает интерпретацию.
- **Тон.** В MVP все провокации `tone: "neutral"`. Поля `"soft"` и `"sharp"` зарезервированы для v2 (настраиваемый тон, см. спеку §11.5).

Если нужно много новых провокаций — ставь задачу «расширить базу провокаций с психологом», не выдумывай формулировки сам.

## Схема `shared-data/formula7/hints.json`

```json
{
  "hints": [
    {
      "id": "h4",
      "keys": ["Ц-6", "П-2", "С-4"],
      "label": "IT / разработка",
      "examples": ["программист", "data scientist", "архитектор ПО"]
    }
  ],
  "schzh_conflicts": [
    {
      "cards": ["СЧЖ-4", "О-1", "О-7"],
      "description": "Богатство vs. честная + тихая работа — сочетание редкое"
    }
  ]
}
```

### Hint

| Поле | Тип | Примечание |
|---|---|---|
| `id` | `string` | уникальный, формат `h<N>` |
| `keys` | `string[]` | коды карточек, ВСЕ должны быть `like` для полного совпадения |
| `label` | `string` | краткое имя направления |
| `examples` | `string[]` | примеры профессий (не рекомендации!) |

### SchzhConflict

| Поле | Тип | Примечание |
|---|---|---|
| `cards` | `string[]` | коды, ВСЕ должны быть `like` для триггера |
| `description` | `string` | формулировка напряжения |

### Правила

- **Hints — это направления, не диагноз.** В UI подаются как «направления, которые напрашиваются», с дисклеймером. Это требование спеки §10.3.
- **Стартовый набор в MVP ~18 hints.** Расширение до 80–100 — отдельная задача с психологом.
- **Алгоритм матчинга** описан в `lib/f7/hints.ts::matchHints`: keep если `coverage >= 1.0` OR (`score >= 2` AND `coverage >= 0.66`). Менять алгоритм — только после обсуждения, он намеренно нестрогий.
- **SchzhConflict — не ошибка пользователя.** UI подаёт как «напряжения», не «противоречия». Спека §17.3.

## Как добавить новую сущность (процесс)

### Новая карточка F7

1. Добавить в `shared-data/formula7/cards.json` в блок `cards[]` с уникальным `code`.
2. Если меняет размер группы — обновить инвариант в `core/tests/test_formula7.py`.
3. Убедиться, что фронт и бэк тесты всё ещё проходят.
4. **Нет необходимости менять TS-код или Python-код** — данные читаются из JSON.

### Новая провокация

1. Добавить в `shared-data/formula7/provocations.json` с уникальным `id`.
2. Если это `pattern`-провокация с новым `kind` триггера — добавить обработчик в `frontend/src/lib/tracker.ts` (и симметрично в `core/src/professyans_core/methods/formula7.py`, если триггеры будут вычисляться server-side).

### Новая hint-сигнатура

1. Добавить в `shared-data/formula7/hints.json::hints[]` с `id: "h<N>"` (следующий номер).
2. Проверить, что `examples[]` — примеры, не «рекомендации».
3. Добавить unit-test в `core/tests/test_formula7.py::test_match_hints_*` если нужно — обычно достаточно существующих regression-тестов.

### Новая методика (F-5, КЧГ, Перекрёсток)

См. `05_ROADMAP_TASKS.md` — там описан полный процесс с атомарными шагами для Cursor.

## Правило парности (перепроверяй на каждом PR)

Если правишь `shared-data/`, проверь:

- [ ] тесты `core/tests/` проходят;
- [ ] тесты `backend/tests/` проходят;
- [ ] `frontend && npm run typecheck` — без ошибок;
- [ ] `frontend && npm run build` — собирается;
- [ ] визуально открыл приложение и прошёл хотя бы одну группу до конца — чтобы убедиться, что фронт не упал из-за недостающего поля.
